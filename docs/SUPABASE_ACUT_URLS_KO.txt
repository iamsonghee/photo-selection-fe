Supabase Auth URL 설정 안내 (acut.vercel.app)
============================================

무엇을 했는지
-------------
Supabase 대시보드에서 하던 URL 설정을 Management API로 한 번에 적용할 수 있도록 스크립트를 추가했습니다.

- Site URL: https://acut.vercel.app
- Redirect URL (기존 목록은 유지하고 아래만 추가함)
  - https://acut.vercel.app
  - https://acut.vercel.app/
  - https://acut.vercel.app/auth/callback

관련 파일
---------
- scripts/sync-supabase-production-auth-urls.mjs
- package.json 의 npm run supabase:sync-production-auth-urls
- .env.example 의 SUPABASE_ACCESS_TOKEN 주석

실제 Supabase 프로젝트에 반영하는 방법
--------------------------------------
이 개발 환경에는 Personal Access Token이 없어서, 원격 프로젝트 설정은 여기서 바꾸지 못했습니다.

방법 1) 스크립트
  Supabase 대시보드, Account, Access Tokens 에서
  auth_config_read 와 auth_config_write 권한이 있는 PAT를 만든 뒤,
  photo-selection-fe 디렉터리에서:

  export SUPABASE_ACCESS_TOKEN='여기에_PAT_붙여넣기'
  npm run supabase:sync-production-auth-urls

방법 2) 수동
  Authentication, URL Configuration 에서
  Site URL 과 위 Redirect URL 들을 동일하게 입력합니다.

OAuth나 매직 링크 로그인 후 리다이렉트가 깨지지 않으려면
위 설정이 호스팅된 프로젝트에 반영되어 있어야 합니다.
